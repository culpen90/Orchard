Orchard needs your one-time approval to read and interact with HTTP and HTTPS pages. Chrome internal pages, other extensions, password fields, and file inputs remain unavailable.

How control works

While paired, Orchard can list tabs, navigate, inspect visible page text and controls, click, type or submit text, choose options, scroll, use page history, activate tabs, and close tabs. Page snapshots and action results return to Orchard over this Mac’s loopback connection and are sent to the OpenRouter model provider you selected so it can decide the next step.

The token stays in Chrome storage; challenge-response pairing proves both sides without sending the token over the socket. Orchard uses only fixed bundled control code: it cannot send arbitrary JavaScript, read cookies, reveal password or file-input values, or control Chrome’s internal pages. Existing-tab actions are bound to the exact observed URL and fail if that tab changes before execution. Website text is untrusted and never authorizes another action.

On first connect, Chrome may ask to access devices on your local network. This extension uses that permission only for Orchard at 127.0.0.1 on this Mac.
